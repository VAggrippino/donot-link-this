Don't Link This!

The Problem

Community services like Facebook and Twitter take plain text as input and turn them into links to relevant content based on commonly used patterns. Three words separated only by dots is treated as a link to a website. An "at sign" followed by a word is treated as a link to a user name or pseudonym. And a "number sign" (#) is treated as a link to a concept stream, also known as a hashtag.

Sometimes, the automatic linking performed by these services is not intended by the author of the content.

The Solution

This tool is an attempt to correct the problem by inserting a zero-width space after certain characters. This character, which is not visible, disrupts the pattern without changing the appearance of the text.

What services does this work with?

Since inserting a non-displayed character invalidates the URL, this should work with any service that automatically transform a text string containing a URL into a clickable link.
